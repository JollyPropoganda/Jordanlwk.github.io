Project Overview

( This Project is a NS derived Project )

Problem statement

With the enormous amount of Full-time national servicemen who enlist in the Army each year it can be really hard to keep track of the amount of exercises each serviceman does, with exercises being a crucial component of the army.

Project scope

Create an app which will allow for an easier tracking of NS servicmen's record of exercise to determine who has and have not been heat acclimatised

    Database works

    For our database we were tasked to integrate our excel datasheet into Appsheet which is a platform which provides codeless app development capabilities.

    Appsheet utitlises "SQL" language to input "what if", "Valid if" criterias.

    Appsheet also includes input for the type of Datafield making it easier to format the data as compared to formatting it in excel.

    The appsheet dashboard boast of SQL functionalities at the check of a box, fields like "Key" and "Type" which usually needs to be coded in as an SQL statement can be now be "ON" just by checking a box.

    This is our Appsheet Database Main page where we can assess all our different Databases.
  • On the left side of the screen, under the "Data" column, are the tabs of all our Databases.
  • At the center of the screen are the Fields for the current Database.

    • These are the different functionalities available within a valuefield in a database:
  • Type Details - enhancements to type i.e (if type is "number" -> minimum / maximum value).
  • Data Validity - Creates conditions to check if data is a valid entry.
  • Auto Compute - Allows for formula to be inputed, formula will be used to compute data.
  • Update Behaviour - Allows value to either be a "Key" value, makes the value editable.
  • Display - Changes the value display name and description.
  • Other Properties - other functionalities to modify data properties such as Searchable/ Scannable/ NFC scannable/ Sensitive data
